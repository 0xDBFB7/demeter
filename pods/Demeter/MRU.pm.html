NAME

Demeter::MRU - Handle lists of recently used file

VERSION

This documentation refers to Demeter version 0.9.26.

DESCRIPTION

This module contains methods for reading from and writing to lists of recently used files.

METHODS

push_mru

Push a file onto the top of the list of recently used files for a file group.

  $atoms_object -> push_mru( "atoms", $input_file);

This pushes $input_file to the head of the list of recent files in the atoms group.

You can maintain any number of groups. For instance, you might have separate groups for data, project files, crystal data files, and so on.

get_mru_list

Return the list of recently used files from a file group:

  my @list_of_files = $atoms_object->get_mru_list("atoms");

or

  my @list_of_files = $atoms_object->get_mru_list(@list_of_goups);

The argument is one or more group names.

This list is actually a list of lists, like so:

  [ [file1, group1],
    [file2, group2],
      ...
    [fileN, groupN],
  ]

where the groups are things like "atoms", "feff" -- that is categories of files used to organize the ini file containing the recent files.

DIAGNOSTICS

None.

CONFIGURATION AND ENVIRONMENT

The recently used file lists are kept in a file called demeter.mru in the dotfile directory, $HOME/.horae on unix and something on WIndows.

See Demeter::Config for a description of the configuration system.

DEPENDENCIES

Demeter's dependencies are in the Build.PL file.

BUGS AND LIMITATIONS

  • List length is 16 items. This should be configurable.

Please report problems to the Ifeffit Mailing List (http://cars9.uchicago.edu/mailman/listinfo/ifeffit/)

Patches are welcome.
