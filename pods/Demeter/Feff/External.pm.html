NAME

Demeter::Feff::External - Import and manipulate external Feff calculations

VERSION

This documentation refers to Demeter version 0.9.26.

SYNOPSIS

  my $feff = Demeter::Feff::External -> new();
  $feff   -> set(workspace=>"temp", screen=>0);
  $feff   -> file('/path/to/feff.inp');
  foreach my $sp (@{$feff->pathlist}) {
    Demeter::Path -> new(sp=>$sp) -> plot('r');
  };

DESCRIPTION

This extension of the Demeter::Feff class allows you to import an externally calculated Feff6 (or Feff7) calculation and interact with it very similarly to a Feff calculation handled in the normal manner in Demeter.

The assumption is that you have a folder somewhere on disk in which you have already made a Feff calculation. Specifically, that folder should be populated by

  • A feff.inp, which can be called something other than feff.inp.

  • The phase.bin file from the calculation.

  • Some number of feffNNNN.dat files from that calculation.

  • Possibly a files.dat and paths.dat files as well, although neither is currently used. These are written normally by Feff and should be found in any folder containing a Feff calculation.

When you set the workspace and file attributes inherited from the normal Demeter::Feff object, the feff.inp file is parsed to set the attributes inherited from Demeter::Feff, the phase.bin file is copied to the workspace, and the feffNNNN.dat files are parsed and made into Demeter::ScatteringPath objects. The ScatteringPath objects are then loaded into the pathlist attribute of this object.

Once all that is done, everything proceeds in the manner of a Demeter-handled Feff calculation. Path objects are created using the ScatteringPath objects generated from the feffNNNN.dat files as the values of the sp attribute (see Demeter::Path). Most methods of the Feff object are available, including find_path and find_all_paths (see Demeter::Feff::Paths).

There are a few differences bewteen a normal Feff calculation and an external one. This object has no pathfinder method at this time, the assumption being that the pathfinder need not be run again. Also the ScatteringPath objects do not retain a memory of the geometries of the degenerate paths nor will the concept of fuzzy degeneracy used in any way.

Aside from being able to import existing Feff calculations, there are two obvious uses for the external Feff object:

  • Import old-style Artemis project files, which contain entire Feff calculations

  • Consider Feff calculations with more than 4-legged paths, something Demeter's pathfinder cannot yet do.

ATTRIBUTES

Along with all the attributes inherited from the Demeter::Feff object, these are added:

folder

The location on disk of the external Feff calculation. This is set automatically once the file attribute is set.

phasebin

The fully resolved file name for the phase.bin file.

filesdat

The fully resolved file name for the files.dat file. This is not currently used.

pathsfile

The fully resolved file name for the paths.dat file from the external Feff calculation. This is not currently used.

Note that the source attribute is set to "external" for a Demeter::Feff::External object.

METHODS

There are no new methods relevant to the user beyond those inherited from the Demeter::Feff object.

CONFIGURATION AND ENVIRONMENT

See Demeter for a description of the configuration system.

DEPENDENCIES

The dependencies of the Demeter system are in the Build.PL file.

BUGS AND LIMITATIONS

  • Import a Feff8 calculation (requires ability to parse a Feff8 input file).

  • Serialization/deserialization is scantily tested.

  • Let pathfinder method actually run Feff's pathfinder.

Please report problems to the Ifeffit Mailing List (http://cars9.uchicago.edu/mailman/listinfo/ifeffit/)

Patches are welcome.
