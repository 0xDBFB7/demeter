NAME

Demeter::UI::Artemis::Import - Import various kinds of data into Artemis

VERSION

This documentation refers to Demeter version 0.9.26.

SYNOPSIS

This module exports the Import method, which is used to import various kinds of data into Artemis.

METHODS

Import

Import a specific data type.

  import($type, $filename);

If the filename is omitted, the user will be prompted for a file using the file selection dialog. The type can be one of:

prj or athena

An Athena project file

chi

A column data file containing chi(k) data. If the data are on a non-standard grid, they will be rebinned onto the proper grid.

dpj

A serialization file from a Fit object. This sort of file is generated from a free-standing script using Demeter.

feff

A feff.inp file, an atoms.inp file, or a CIF file.

external

An entire Feff calculation. Note that a feff.inp file by itself can be imported using

old

An project file from the Tk version of Artemis.

prjrecord

Post a dialog prompting for a record in an Athena project file.

DEPENDENCIES

Demeter's dependencies are in the Build.PL file.

BUGS AND LIMITATIONS

  • Not handling local parameters from feffit.inp files

  • While single-data-set fits with multiple k-weights does work correctly, MDS+MKW fits will not be imported properly from a feffit.inp file

Please report problems to the Ifeffit Mailing List (http://cars9.uchicago.edu/mailman/listinfo/ifeffit/)

Patches are welcome.
