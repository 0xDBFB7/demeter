NAME

Demeter::Data::Defaults - Resolve default parameter values

VERSION

This documentation refers to Demeter version 0.9.19.

DESCRIPTION

This role of Demeter::Data contains methods resolving default parameter values from the contents of the data. These rarely need to be called in a program, but they need to be documented so that values in configuration files can be understood.

METHODS

resolve_defaults

This method dispatches to the other methods of tyhis module depending on whether the data are mu(E) or chi(k).

resolve_e0_fraction

This method sanitizes the bkg_e0_fraction parameter. Values greater than 1 are set to 1. Values of 0 or less are set to 0.5.

resolve_clamps

This method converts the words "none", "slight", "weak", "medium", "strong", or "rigid" to their numeric values. See the clamp configuration group for tuning the translation between string and numeric clamp values.

resolve_pre

This method resolves the default values for bkg_pre1 and bkg_pre2 to values appropriate to the data. If the pre1 value is set to 0, bkg_pre1 is set to evaluate the second data point. If either the pre1 or pre2 values is positive, it is set to evaluate that far above the first data point. Numbers less than 1 are interpreted as keV. bkg_pre1 and bkg_pre2 are sorted such that bkg_pre1 is less than bkg_pre2. See the bkg configuration group for tuning these defaults.

resolve_nor

This method resolves the default values for bkg_nor1 and bkg_nor2 to values appropriate to the data. If the nor2 value is set to 0, bkg_nor2 is set to evaluate the last data point. If either the nor1 or nor2 values is negative, it is set to evaluate that far below the last data point. Numbers less than 5 are interpreted as keV. bkg_nor1 and bkg_nor2 are sorted such that bkg_nor1 is less than bkg_nor2. See the bkg configuration group for tuning these defaults.

resolve_spl

This method resolves the default values for bkg_spl1 and bkg_spl2 to values appropriate to the data. If the spl2 value is set to 0, bkg_spl2 is set to evaluate the last data point. If either the spl1 or spl2 values is negative, it is set to evaluate that far below the last data point. bkg_spl1 and bkg_spl2 are sorted such that bkg_spl1 is less than bkg_spl2. See the bkg configuration group for tuning these defaults.

resolve_krange_xmu

This method resolves the default values for fft_kmin and fft_kmax to values appropriate to the mu(E) data. If the kmax value is set to 0, fft_kmax is set to evaluate the last data point. If either the kmin or kmax values is negative, it is set to evaluate that far below the last data point. fft_kmin and fft_kmax are sorted such that fft_kmin is less than fft_kmax. See the fft configuration group for tuning these defaults.

resolve_krange_chi

This method resolves the default values for fft_kmin and fft_kmax to values appropriate to the chi(k) data (that is, data that are originally imported as chi(k)). If the kmax value is set to 0, fft_kmax is set to evaluate the last data point. If either the kmin or kmax values is negative, it is set to evaluate that far below the last data point. fft_kmin and fft_kmax are sorted such that fft_kmin is less than fft_kmax. See the fft configuration group for tuning these defaults.
