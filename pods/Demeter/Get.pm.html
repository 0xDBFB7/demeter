NAME

Demeter::Get - Choke point for probing Ifeffit, Larch, or other backends

VERSION

This documentation refers to Demeter version 0.9.24.

SYNOPSIS 

  $number = $object -> fetch_scalar('foo');
  $string = $object -> fetch_string('bar');
  @array  = $object -> fetch_array('blah.x');

  $object -> place_scalar('foo', 5);
  $object -> place_string('bar', 'Fred');
  $object -> place_array('blah.x', \@list);

DESCRIPTION

This module provides a single choke point for retrieving data from Ifeffit, Larch, or other data processing backeeeends. Based on the value of the Mode objects "template_process" attribute, the correct thing will be done to obtain scalar, string, array, or other data from the backend.

METHODS

Methods for getting data

fetch_scalar

Return a scalar value from a named variable in the backend's memory.

fetch_string

Return a string value from a named variable in the backend's memory.

fetch_array

Return an array value from a named variable in the backend's memory. Note that this returns an array, not an array reference.

Methods for putting data

place_scalar

Push a scalar value to a named variable the backend.

place_string

Push a string value to a named variable the backend.

place_array

Push an array value to a named variable the backend. Note that this takes an array reference as its argument.
